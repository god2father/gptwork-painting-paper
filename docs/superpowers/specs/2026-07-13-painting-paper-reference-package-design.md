# GPT Work 前置项目包设计

## 目标

建立一个可直接交给 GPT Work 的本地目录，包含参考视频固定路径、自动下载与预处理脚本、任务指令、视觉规范、验收条件和配置模板。

## 输入

- 抖音分享地址。
- 下载后的 `reference/reference-video.mp4`。
- GPT Work 后续识别并获取的公共领域原始名画。

## 目录职责

- `reference/`：原视频、提取帧、音频和来源信息。
- `assets/`：原始名画、分层素材和纹理。
- `docs/`：分析、规范、审查和验收文档。
- `manifests/`：时间线与作品场景配置。
- `scripts/`：下载、预处理和完整性检查。
- `src/`、`tests/`：GPT Work 后续生成的网站源码与测试。

## 失败处理

抖音可能因登录、Cookie、地区或自动化风控阻止下载。项目包不得伪造 MP4；下载失败时保留 URL、作品 ID、手动放置路径和可重复执行的脚本，其余不依赖视频的准备工作仍然完成。

## 成功标准

用户解压后，只需准备 `reference/reference-video.mp4`，随后在 GPT Work 中打开目录并粘贴 `WORK_PROMPT.md`，即可开始分析和开发。
