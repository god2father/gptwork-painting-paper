# 《戴珍珠耳环的少女》互动剪纸舞台设计

## 1. 目标与范围

第一版网站只展示一幅作品《戴珍珠耳环的少女》。用户进入页面时先看到完整组合的名画，可以通过滚动、自动播放或时间轴把画面拆解为 12 个纸张图层，再还原到完整构图。

本阶段必须提供：自动播放、暂停、重播、还原、时间轴跳转、滚动拆解、克制的鼠标视差、点击图层查看信息、键盘操作、减少动画降级、桌面和移动端布局，以及实际视觉基准截图。

本阶段不做多作品切换、后台编辑器、自由拖拽、用户上传、音乐、视频背景或服务端接口。作品选择入口保留清晰边界，但不显示无效的第二幅作品。

## 2. 视觉与页面结构

页面使用暖米白纸张底色、深褐文字和细微纤维感，名画本身位于深色竖幅舞台。UI 保持中性，不使用科技蓝渐变、玻璃拟态或塑料 3D 材质。

桌面 1920×1080 布局：

- 顶部左侧显示作品编号、中文标题和一句引导语。
- 中央为 4:5 竖幅舞台，完整显示 1536×1920 画布。
- 右侧为图层检查器；未选中图层时展示操作提示，选中后显示名称、说明和当前层级。
- 底部为固定但不遮挡作品的控制条：播放/暂停、重播、还原、进度滑块和百分比。
- 页面提供约 260vh 的滚动区，舞台在拆解段保持粘性定位。

移动端使用单列结构：标题在上、舞台居中、控制条换行，图层检查器变为舞台下方的信息卡。页面不得横向溢出，所有按钮保持至少 44×44 CSS 像素的点击区域。

## 3. 交互状态与运动

唯一规范化进度为 `0..1`：

- `0`：完整组合状态。
- `0.05..0.85`：12 个图层按照 JSON 中的开始时间、持续时间和缓动错峰展开。
- `0.85..1`：保持拆解状态，方便点击查看。

滚动、自动播放和时间轴滑块都驱动同一个进度，不建立三套动画。自动播放时滚动只改变页面位置，不抢夺播放进度；用户拖动滑块、点击还原或开始主动滚动时，自动播放停止。

每个图层的组合态、拆解态、动画区间、层级、视差强度和说明全部来自作品 JSON。通用舞台组件不得出现 `painting-01`、人物部位名称或具体像素位置。

鼠标视差作用于 GSAP 动画之外的内层包装，避免覆盖拆解变换；离开舞台时平滑回到零。触摸设备不启用指针视差。

点击可见图层后设置选中态，在检查器中显示信息，并用克制的暖白描边提示当前层。按 `Escape` 清除选择。

`prefers-reduced-motion: reduce` 时禁用自动播放、滚动缓动、视差、回弹和旋转；滚动进度仍可直接切换组合/拆解状态，时间轴和按钮继续可用。

## 4. 配置模型

作品配置位于 `manifests/paintings/painting-01.json`，并由 `manifests/paintings/painting.schema.json` 校验。核心结构包含：

- `id`、`title`、`subtitle`、`duration`。
- `canvas.width`、`canvas.height`。
- `background.src`、`background.alt`。
- `layers[]`：`id`、`name`、`src`、`alt`、`description`、`z`。
- `layers[].collapsed`：`x`、`y`、`scale`、`rotation`、`opacity`。
- `layers[].expanded`：`x`、`y`、`scale`、`rotation`、`opacity`。
- `layers[].animation`：`start`、`duration`、`ease`。
- `layers[].parallax`：`x`、`y`。

配置坐标以作品画布像素表示，运行时统一换算为舞台显示尺寸。资源路径使用 `/assets/...` 形式，通过 Vite 的 `import.meta.glob` 资源映射转换为构建后的 URL，不复制 `assets/` 目录，也不把文件名硬编码到组件中。

## 5. 前端边界

- `src/features/stage/`：通用分层舞台、图层节点和指针视差。
- `src/features/timeline/`：播放控制、滑块和滚动进度连接。
- `src/features/inspector/`：图层信息与选择状态展示。
- `src/features/gallery/`：第一版只负责作品标题与未来作品入口边界。
- `src/lib/scene/`：配置类型、校验、资源 URL 解析和画布坐标换算。
- `src/lib/motion/`：从配置构建唯一 GSAP 时间轴，不引用具体作品。
- `src/stores/`：Pinia 保存进度、播放状态和选中图层 ID。
- `src/types/`：共享的作品、图层和变换类型。

页面根组件只负责组合上述功能，不直接操作具体图层 DOM。GSAP 时间轴由舞台挂载后创建并在卸载时清理；ScrollTrigger 同样必须清理，避免开发热更新时重复注册。

## 6. 异常与可访问性

配置解析失败、资源缺失或图层数组为空时，页面显示清晰的中文错误卡和重新加载按钮，不能停留在空白屏幕。单个图片加载失败时保留舞台尺寸，并在检查器中标记资源错误。

控制按钮使用原生 `button`，滑块使用 `input[type=range]`，图层使用可聚焦按钮语义。所有图标按钮有中文可访问名称；键盘焦点清晰可见；舞台有描述当前作品和交互方式的可访问文本。

播放状态使用 `aria-pressed`，进度使用可读百分比，图层检查器使用 `aria-live="polite"` 报告选择变化。装饰性纸张纹理不暴露给辅助技术。

## 7. 测试与完成标准

Vitest 覆盖：

- 配置结构和 12 个图层资源解析。
- 画布坐标到 CSS 位移的换算。
- 进度钳制、播放/暂停/还原状态。
- 减少动画模式下的直接状态切换。

Playwright 覆盖：

- 首屏显示完整作品。
- 播放、暂停、重播、还原和时间轴跳转。
- 滚动后图层发生位移。
- 点击图层后检查器更新，`Escape` 可清除。
- 1920×1080 和 390×844 均无横向溢出，控制按钮可点击。
- 生成桌面组合态、桌面拆解态和移动端三张实际截图到 `tests/visual/actual/`。

只有 `npm run typecheck`、`npm run test`、`npm run test:e2e`、`npm run build` 全部通过，三张视觉截图已打开检查，且 README 包含安装、运行、素材替换和新增作品方法时，第一幅作品才算完成。

## 8. 已确认决策

- 采用方案 A：全屏博物馆式粘性舞台。
- 第一版只实现一幅作品。
- 使用 Vue 3、Vite、TypeScript、GSAP、ScrollTrigger、Pinia、Vitest 和 Playwright。
- 不新增 UI 库、状态库、CSS 框架或资源复制插件。
- 所有作品特定的位置、运动和文案进入 JSON 配置。
- 参考视频只用于效果理解，网站不播放或发布其视频与音乐。
