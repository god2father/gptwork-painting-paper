# 《戴珍珠耳环的少女》纸艺工作台改版设计

## 1. 改版目标

把当前“三栏博物馆查看器”改造成一个完整的俯拍纸艺工作台场景。页面首先应像一件可观看的纸艺作品，其次才是一个带控制功能的网站。

本轮保留已有的 Vue、Pinia、GSAP、统一进度和 JSON 配置基础，重新制作场景素材、作品分层、页面构图和动画编排。第一版仍只展示《戴珍珠耳环的少女》，不增加第二幅作品、音乐、视频背景、手部表演或自由拖拽。

## 2. 视觉方向

采用“艺术家的俯拍工作台”方向：

- 视口是一张横向工作台，而不是网页分栏。
- 桌面使用 GPT 生成的原创暖棕色旧木纹背景，边缘自然摆放裁切纸、胶带、铅笔和少量纸屑；不得出现手、文字、商标或抢眼杂物。
- 中央是一张带不规则毛边的暖白手工纸，纸上嵌入 4:5 肖像阴影盒。
- 作品画框占桌面端视口宽度约 36% 到 44%，环境留白参与构图。
- 光源固定来自左上方，所有纸片、画框和装饰物的阴影向右下方延伸。
- 颜色以木色、暖白、深褐、群青和低饱和赭黄为主；不使用纯黑控制条、科技蓝或玻璃拟态。

静止首屏必须能够独立作为一张完整的艺术海报成立。

## 3. 场景结构

页面由五个视觉层级组成：

1. `workspace`：GPT 生成的完整俯拍木桌和边缘工具背景。
2. `paper-bed`：中央毛边手工纸、胶带和局部纸屑。
3. `portrait-frame`：带纸张厚度和内凹阴影的 4:5 画框。
4. `portrait-layers`：可独立运动的肖像纸片。
5. `interface`：左上作品纸签、底部轻量控制和按需出现的图层注释纸条。

桌面、底纸和画框共同构成主舞台。标题和检查器不再占据固定列宽，避免和少女面部争夺视觉中心。

## 4. 肖像分层

将当前 12 个碎片层改为 7 个具有物理和构图意义的语义层：

1. 深色背景。
2. 褐色肩部与衣身。
3. 颈部、耳朵和脸部主体。
4. 蓝色头巾。
5. 黄色包布和垂落巾带。
6. 双眼与眉部微动层。
7. 珍珠与高光层。

脸部亮面和暗面不再作为独立飞散纸片，而是合并在脸部主体中。每个透明图层需要统一的暖白窄纸边、细微纤维和右下柔影。完全组合时缝隙应自然；完全展开时仍能清楚识别少女的姿态、脸部轮廓和视线。

新素材从 GPT 生成的统一主图派生，保留生成记录、提示词、日期和本地文件映射。禁止从参考视频截图中抠取发布素材。

## 5. 动画叙事

所有动作继续由 `0..1` 的单一进度驱动，划分为五幕：

### 5.1 纸幕揭开（0–0.12）

上下两张带毛边的暖白纸幕向外滑开，露出工作台。运动使用短距离、低回弹的纸张惯性，不出现数字化淡入。

### 5.2 画框入场（0.12–0.28）

肖像画框从桌面下方轻轻滑入，带约 1.5° 的角度修正和逐渐收紧的右下阴影。画框落位后停顿，建立观看焦点。

### 5.3 镜头靠近（0.28–0.42）

整个工作台舞台缓慢缩放，画框从环境中的物件变成主角。背景工具相对退向边缘，肖像内部不拆散。

### 5.4 纸层呼吸式展开（0.42–0.82）

图层按照背景、衣身、脸部、头巾、眼睛、珍珠的构图关系错峰抬升。主要变化是 Z 轴深度、8–36px 的局部错位、0.5°–2.5° 的小角度扇形展开和阴影距离变化，不允许大幅横向飞散。

展开过程中：

- 脸部主体保持稳定，眼睛不与脸部远距离分离。
- 蓝黄头巾可以产生最明显的层差，但仍需保持头部轮廓。
- 珍珠最后抬升，出现一次克制的高光变化。
- 任意进度下肖像都保持可识别。

### 5.5 观察与回收（0.82–1）

图层保持可点击的展开状态。触发还原或重新播放时，纸片按照相反层级逐层落回，阴影同步收紧。自动播放到终点时不自动跳回起点。

`prefers-reduced-motion` 下取消纸幕惯性、镜头缩放、回弹和指针视差，只在组合态与轻微展开态之间直接切换。

## 6. 交互与界面

- 左上角使用一张小型作品纸签，显示编号和中文标题；不再使用占据整列的大标题。
- 底部控制器改为暖白纸条或深褐细线刻度，只保留播放/暂停、进度和一个次级菜单入口。
- 重播和还原进入次级操作区域，不再使用三个等权按钮。
- 控制器在键盘聚焦、鼠标靠近或播放时显示；静止后降低透明度，但不得影响键盘访问。
- 图层说明默认隐藏。点击纸片后，在画框附近滑出一张小纸签，显示名称和一句构图说明。
- `Escape` 清除选择，点击另一层直接切换说明。
- 图层点击只增加 8–16px 的视觉抬升和阴影，不改变时间轴进度。
- 指针视差只影响工作台、画框、头巾尾部和珍珠，五官主体不随鼠标漂移。

所有按钮保持至少 44×44 CSS 像素点击区域，具有可见焦点和中文可访问名称。

## 7. 配置和组件边界

作品 JSON 扩展但继续保持配置驱动：

- `environment`：工作台、底纸、纸幕和画框素材。
- `camera`：各时间段的舞台缩放、平移和缓动。
- `groups`：脸部、头巾等父子关系，防止子层脱离主体。
- `layers[].expanded`：增加 `z`、阴影强度和局部抬升参数。
- `chapters`：五幕的开始、结束和标签。

核心舞台组件不得出现作品名称或作品特定坐标。作品层的父子关系、顺序、移动范围和说明全部来自配置。

建议组件边界：

- `WorkspaceStage.vue`：组织工作台、底纸、画框和相机层。
- `PortraitFrame.vue`：处理画框裁切、深度和作品舞台。
- `StageLayer.vue`：渲染单个可交互纸片。
- `PaperLabel.vue`：复用标题纸签和图层说明纸签。
- `TimelineControls.vue`：轻量控制器。
- `buildTimeline.ts`：从配置生成纸幕、镜头、画框和图层时间轴。

## 8. 响应式策略

桌面以 1920×1080 为目标。移动端保留工作台氛围，但减少边缘工具数量，画框宽度提高到视口的 72%–82%。标题纸签位于左上，图层说明位于画框下方，控制器贴近底部安全区。

移动端关闭指针视差和较大的 Z 轴扇形，只保留画框入场、镜头靠近和轻微分层抬升。页面不得产生横向滚动。

## 9. 验收标准

视觉验收：

- 首屏不看控件也像一张完整的纸艺工作台作品。
- 画框、底纸和桌面形成明确的主次关系。
- 所有阴影方向一致，纸片有可见但克制的厚度。
- 完全展开时少女仍然可识别，没有五官碎片无意义漂浮。
- UI 视觉重量明显低于作品，黑色工具栏不再存在。
- 1920×1080 与 390×844 都没有横向溢出。

交互验收：

- 播放、暂停、拖动、还原和滚动继续驱动同一进度。
- 点击图层、切换图层和 `Escape` 清除均可用。
- 减少动画模式提供稳定降级。
- 资源失败显示清晰错误，不出现空白舞台。

工程验收：

- 类型检查、单测、端到端测试和生产构建全部通过。
- 生成桌面首屏、桌面展开态和移动端三张实际截图并逐张检查。
- 生成素材的来源、提示词和处理流程有记录。

## 10. 保留与废弃

保留：Vue/Pinia/GSAP 架构、单一进度、配置校验、资源解析、交互状态、基础可访问性和现有测试思路。

废弃：三栏博物馆布局、常驻右侧检查器、黑色固定工具条、12 个任意碎片的大幅飞散、脸部亮暗面独立拆出，以及全画面统一鼠标视差。
